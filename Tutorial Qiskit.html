Primeiros Circuitos Quânticos¶

Este tutorial no ter muita coisa, s de forma a passar a informao mais bsica sobre como trabalhar com o qiskit.

Em computao quntica, comeamos tambm por trabalhar com portas lgicas; no entanto, enquanto que classicamente podemos ter 2 bits como input e s um como output, em quntica, queremos que todas as nossas portas sejam reversveis; logo, se temos um input de n qubits, temos output de n qubits. MAIS: queremos que, para um gate $\hat{G}$, passar por ele duas vezes o mesmo que no fazer nada (tipo um NOT gate em computao clssica); ou seja, $\hat{G}\hat{G}=Id$.

Algumas portas lgicas comuns so: $X$, $Y$, $Z$ e $H$. As primeiras 3 so as matrizes de Pauli. A ltima a de Hadamard, que introduz sobreposio. Vamos ver a porta de Hadamard muitas vezes, j que sobreposio chave em computao quntica. Se queremos investigar o comportamento de uma funo $f(x)$, classicamente, podemos comear por atribuir vrios valores a $x$; porm, em quntica, podemos usar sobreposio, onde, na medio final, todos os valores de $x$ so considerados num instante.

Algumas notas a ter em mente

  • Ao dar um vetor ao teu sistema: lembra que tem de estar SEMPRE normalizado (soma dos quadrados igual a $1$).
  • Simuladores aqui no qiskit permitem-nos ver os estados dos qubits (para aprender), mas num computador quntico a srio, isso no possvel.
  • Podemos pensar em 2 estados qunticos imaginando a esfera de Bloch. A, temos que $|q \rangle = \cos \frac{\theta}{2} |0 \rangle + e^{i\phi} \sin \frac{\theta}{2} |1 \rangle$; no entanto, lembra que fase global no importa na medio...

def sim_and_count(circuito, backend_usado='qasm_simulator'):
    """Correr um circuito num backend ou simulador e contar os resultados obtidos"""
    backend = Aer.get_backend(backend_usado)  # para correr num computador quntico, basta trocar para um backend a srio
    results = execute(circuito, backend=backend, shots=100).result()
    counts = results.get_counts()
    return counts

counts = sim_and_count(qc)
print(counts)

qc_ha = QuantumCircuit(4,2)
# encode inputs in qubits 0 and 1
qc_ha.x(0) # For a=0, remove the this line. For a=1, leave it.
qc_ha.x(1) # For b=0, remove the this line. For b=1, leave it.
qc_ha.barrier()
# use cnots to write the XOR of the inputs on qubit 2
qc_ha.cx(0,2)
qc_ha.cx(1,2)
# use ccx to write the AND of the inputs on qubit 3 (this gate is also called the "Toffoli")
qc_ha.ccx(0,1,3)
qc_ha.barrier()
# extract outputs
qc_ha.measure(2,0) # extract XOR value
qc_ha.measure(3,1) # extract AND value

qc_ha.draw()
